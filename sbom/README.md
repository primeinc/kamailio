# Software Bill of Materials (SBOM)

This directory contains Software Bill of Materials (SBOM) files for the Kamailio project.

## SBOM Files

SBOM files are automatically generated by the CI/CD pipeline and include:

- **kamailio-repo.spdx.json** - SPDX format SBOM for the entire repository
- **kamailio-repo.cyclonedx.json** - CycloneDX format SBOM for the entire repository  
- **kamailio-source.spdx.json** - SPDX format SBOM for source code only
- **kamailio-source.cyclonedx.json** - CycloneDX format SBOM for source code only
- **kamailio-container.spdx.json** - SPDX format SBOM for container images (if available)
- **kamailio-container.cyclonedx.json** - CycloneDX format SBOM for container images (if available)
- **dependencies.md** - Human-readable dependency summary

## Generation

SBOM files are generated using [Syft](https://github.com/anchore/syft) and are updated:
- On every push to main/master branches
- On pull requests
- Weekly via scheduled workflow
- On manual workflow dispatch

## Usage

These SBOM files can be used for:
- Security vulnerability scanning
- License compliance analysis  
- Supply chain risk assessment
- Dependency tracking and management

## Formats

- **SPDX**: Software Package Data Exchange format (ISO standard)
- **CycloneDX**: Lightweight SBOM format focused on security use cases

For more information on SBOM standards, see:
- [SPDX Specification](https://spdx.github.io/spdx-spec/)
- [CycloneDX Specification](https://cyclonedx.org/specification/overview/)
